# AI-bum (실버 케어 액자) 프로젝트 진행 현황 요약서

본 문서는 `AI-bum` 프로젝트의 현재까지 개발된 마일스톤과 코드 구조를 다른 AI(또는 작업자)에게 전달하여 문맥을 연동하기 위해 작성되었습니다.

---

## 1. 프로젝트 개요
**AI-bum**은 비전 인식 기술과 대형 언어 모델(LLM)을 결합하여, 어르신이 액자(기기) 앞에 다가왔을 때 얼굴을 감지하고 다정한 손주 모드로 음성 대화를 나누며 상태(복약 등)를 체크하는 스마트 실버 케어 시스템입니다.

## 2. 현재 시스템 아키텍처 및 스택 (Backend)
현재 프로토타입은 노트북(웹캠+마이크) 환경에서 테스트 중이며, 핵심 스택은 다음과 같습니다.
*   **프레임워크**: `FastAPI` (웹소켓 기반 비동기 프론트엔드 통신용)
*   **비전 인식**: `OpenCV` (DNN 모듈: `res10_300x300_ssd_iter_140000.caffemodel`)
*   **음성 인식 (STT)**: `SpeechRecognition`, `PyAudio` (마이크 실시간 입력)
*   **언어 모델 (LLM)**: `google-generativeai` (Gemini 1.5 Flash API 적용)
*   **음성 합성 (TTS)**: `gTTS` 및 `pygame` (상태 방해 없는 백그라운드 재생)

---

## 3. 핵심 구현 내용 상세

### A. 공통 설정 파일 분리 (`config.py`)
기존에 `main.py`에 하드코딩 되어있던 각종 상수(카메라 인덱스, 프레임 주기, 판정 민감도 기준 등)와 환경 변수(`GEMINI_API_KEY`)를 별도의 모듈로 분리하여 관리 효율성을 높였습니다.

### B. 비전 기반 상태 머신 (`main.py` - FaceDetector)
DNN 기반으로 비디오 프레임을 분석하여 얼굴을 감지하며, 3가지 주요 상태(State) 로직을 갖습니다.
1.  **`idle` (대기)**: 화면에 아무도 없을 때의 기본 상태.
2.  **`greeting` (인사)**: 3프레임 연속 얼굴 감지 시 진입하는 상태. **(이 상태로 전이될 때 음성 대화 에이전트가 트리거됩니다.)** 5초 유지됩니다.
3.  **`active` (활성)**: 인사가 끝나고 콘텐츠(액자)를 보여주는 상태. 이 상태에서 얼굴 미감지 시간이 30초가 지나면 다시 `idle` 상태로 복귀합니다.

### C. 대화형 AI 손주 모듈 (`voice_agent.py` - VoiceAgent)
독립적으로 실행 가능한 대화형 서비스 로직을 완성했습니다. 주요 특징은 다음과 같습니다.
*   **페르소나 프롬프팅**: *"너는 홀로 계신 할머니/할아버지를 챙기는 다정한 손주야. 말투는 경북 사투리를 자연스럽게 섞어서 써줘."* 라는 시스템 프롬프팅을 적용하여 따뜻한 사용자 경험을 제공합니다.
*   **키워드 감지 및 예외 처리 로직**: STT를 통해 변환된 사용자의 텍스트 중 **"약"**, **"먹었(무긋)"** 등의 키워드가 감지되면 `is_pill_taken = True` 라는 내부 변수 상태를 변경합니다. 이때는 Gemini API 호출을 생략하고 하드코딩된 '폭풍 칭찬 음성'을 즉각적으로 송출하여 빠르고 긍정적인 피드백을 줍니다.
*   **무한 대기 방지**: 마이크 입력에 `timeout` 과 `phrase_time_limit` 을 부여하여 어르신이 아무 말을 하지 않더라도 루프가 갇히지 않도록 구성했습니다.

### D. ⚡ 비전-대화 '브릿지(Bridge)' 연동
가장 중요한 연결 부분입니다. `main.py`의 `_transition` 메서드(상태 전환기)에 훅(Hook)을 걸어 두 모듈을 연결했습니다.
*   비전 코드에서 상태가 `"greeting"`으로 전환될 때 -> `VoiceAgent` 인스턴스를 생성하고 `.start_conversation()`을 호출합니다.
*   대화 모듈은 OpenCV의 `while` 비전 루프를 블로킹(멈춤)하지 않도록 내부적으로 `threading.Thread`를 사용해 백그라운드에서 구동됩니다.
*   어르신이 자리를 떠나 30초가 경과해 `"idle"` 상태로 변경될 때 -> `.stop_conversation()`을 호출하여 마이크를 끄고 대화를 종료시킵니다.
