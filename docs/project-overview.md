# Senior Smile - AI 시니어 케어 디지털 액자

## 1. 프로젝트 소개

**Senior Smile**은 어르신의 일상을 돌보는 AI 기반 디지털 액자 시스템입니다.

웹캠으로 어르신의 얼굴을 감지하면 AI 손주 캐릭터가 음성으로 대화하고, 보호자가 보낸 가족 사진을 슬라이드쇼로 보여줍니다. 복약 알림, 긴급 상황 감지, 감정 분석까지 수행하여 보호자에게 실시간 알림과 리포트를 제공합니다.

### 핵심 가치
- **어르신**: 외로움 해소, 가족과의 소통, 건강 관리
- **보호자**: 원격 모니터링, 안심, 양방향 소통

### 기술 스택
| 구분 | 기술 |
|------|------|
| 프론트엔드 (시니어 앱) | React 19 + Vite 8 |
| 백엔드 | FastAPI + Python 3.13 |
| 얼굴 감지 | OpenCV DNN (Caffe 모델) |
| 음성 인식 (STT) | Google Speech Recognition API (무료, 키 불필요) |
| 음성 합성 (TTS) | OpenAI TTS API (tts-1, nova) |
| 대화 AI | OpenAI GPT-4o-mini (7살 손주 페르소나) |
| 데이터베이스 | Firebase Firestore |
| 푸시 알림 | Firebase Cloud Messaging (FCM) |
| 날씨 | OpenWeatherMap API |
| 인증 | JWT + bcrypt |
| 배포 대상 | Raspberry Pi 5 (예정) |

## 4. 구현 기능 목록

### 4.1 얼굴 감지 시스템
| 항목 | 내용 |
|------|------|
| 모델 | OpenCV DNN (Caffe SSD, 300x300) |
| 신뢰도 임계값 | 0.55 |
| 감지 확인 | 3프레임 연속 감지 후 세션 시작 |
| MJPEG 스트림 | `/video` 엔드포인트로 실시간 영상 제공 |
| 카메라 복구 | 30프레임 연속 실패 시 자동 재연결 |

### 4.2 호출어 기반 음성 대화 (STT/TTS)
| 항목 | 내용 |
|------|------|
| 호출어 | **"앨범아"** (+ 발음 변형: "앨범 아", "엘범아", "앨버마") |
| STT | Google Speech Recognition API (한국어, 무료, 키 불필요) |
| TTS | OpenAI TTS-1 (nova 음성, 속도 1.05) |
| 대화 AI | GPT-4o-mini (7살 손주 페르소나, 일상 대화 지원) |
| 대화 기록 | 최근 5개 메시지 컨텍스트 유지 |
| 에코 방지 | AI 발화 후 0.5초 대기 |
| 에너지 임계값 | 1500 (기본 300 대비 5배, 배경 소음 필터링) |
| 마이크 재연결 | 연결 끊김 시 최대 5회 자동 재시도 |
| TTS 파일 경로 | 절대경로 사용 (`sounds/temp_voice.mp3`) |

#### 호출어 동작 흐름
```
얼굴 감지 → 마이크 ON (호출어 대기)
  │
  ├─ TV 소리, 다른 사람 목소리 → 무시
  ├─ "앨범아" → "네, 말씀하세요!" → 대화 모드 진입
  ├─ "앨범아 오늘 날씨 어때?" → 즉시 대화 처리
  └─ "아파" / "살려줘" → 긴급 감지 (호출어 없이도 항상 동작)

대화 모드:
  ├─ 일반 대화 (GPT 자유 대화, 일상 챗봇)
  ├─ 약 복용 / 식사 / 답장 / 사진 수락 등
  ├─ "그만" / "잘 자" → 대기 복귀
  └─ 10초 무응답 → 자동 대기 복귀
```

### 4.3 사진 슬라이드쇼
| 항목 | 내용 |
|------|------|
| 사진 소스 | ai-bum 백엔드 (`/api/photos`) + 로컬 사진 |
| 전환 간격 | 8초마다 자동 전환 |
| 폴링 | 60초마다 새 사진 확인 |
| 기본 슬라이드 | 사진 없을 때 이모지 + 메시지 4장 |
| 새 사진 알림 | Firebase 실시간 감지 → AI가 어르신에게 알림 → 수락 시 표시 |
| 자동 닫기 | 새 사진 20초 표시 후 자동 복귀 + 타이머 바 UI |

### 4.4 복약 알림
| 항목 | 내용 |
|------|------|
| 알림 시간 | config 설정 (기본 09:00) |
| 스케줄러 | APScheduler (cron) |
| 복용 감지 | "약 먹었어" 키워드 인식 |
| 미복용 경고 | 알림 후 10분 미확인 시 보호자 푸시 알림 |
| 응답 음성 | 복용 확인 시 칭찬 MP3 + 물 마시기 안내 TTS |

### 4.5 긴급 상황 감지
| 항목 | 내용 |
|------|------|
| 감지 키워드 | "아파", "아프", "쓰러", "살려", "도와", "어지러", "못 움직", "119", "응급" |
| 호출어 필요 | **불필요 (항상 감지)** — 안전 최우선 |
| 동작 | 즉시 TTS 응답 ("보호자에게 알릴게요!") + 대화 유지 |
| 보호자 알림 | 긴급 푸시 알림 (쿨다운 무시, 즉시 발송) |
| 프론트엔드 | 빨간 긴급 배너 표시 (펄스 애니메이션) |

### 4.6 푸시 알림 (FCM)
| 이벤트 | 알림 내용 |
|--------|----------|
| 어르신 감지 | "어르신이 액자 앞에 오셨어요" |
| 대화 종료 | "N분간 대화 + 감정 리포트" |
| 약 복용 확인 | "어르신이 HH:MM에 약을 드셨어요" |
| 약 미복용 경고 | "약 복용이 확인되지 않았어요" |
| 사진 확인 | "어르신이 사진을 보셨어요!" |
| 긴급 상황 | "어르신이 도움을 요청하고 있어요!" |
| 음성 답장 | "어르신이 음성 답장을 보내셨어요" |

### 4.7 디바이스 페어링
| 항목 | 내용 |
|------|------|
| 디바이스 ID | 최초 부팅 시 자동 생성 (`frame-xxxxxxxx`) |
| 페어링 코드 | 6자리 숫자, 5분 유효, 1회 사용 후 폐기 |
| 매칭 구조 | 시니어 1명 : 보호자 N명 (family 그룹) |
| 저장소 | Firebase Firestore (devices / families / users) |
| UI | IdleScreen에서 미페어링 시 코드 자동 표시 |

### 4.8 보호자 대시보드
| 기능 | 내용 |
|------|------|
| 접속 URL | `http://localhost:8000/dashboard` |
| 인증 | 회원가입/로그인 필수 (JWT) |
| 통계 카드 | 총 대화 세션, 긴급 상황, 약 복용, 평균 대화량 |
| 감정 트렌드 | 일별 기분 점수 그래프 (Chart.js, 최근 14일) |
| 대화 기록 | 세션별 대화 내역 펼쳐보기 + 감정 리포트 |
| 음성 메시지 | 보호자 녹음 → 어르신 재생 / 어르신 답장 재생 |
| 루틴 현황 | 오늘의 일일 루틴 시간표 표시 |

### 4.9 음성 메시지 (양방향)
| 방향 | 흐름 |
|------|------|
| 보호자 → 어르신 | 대시보드 녹음 → 업로드 → 액자 자동 재생 (호출어 불필요) |
| 어르신 → 보호자 | "답장할래" 키워드 → 15초 녹음 → 저장 → 보호자 푸시 알림 |

### 4.10 일일 루틴 스케줄러
| 시간 | 유형 | 멘트 |
|------|------|------|
| 07:00 | 기상 | "좋은 아침이에요! 오늘도 건강한 하루 보내세요." |
| 09:00 | 복약 | (기존 복약 스케줄러 동작) |
| 12:00 | 점심 | "점심 시간이에요! 맛있는 거 드세요." |
| 15:00 | 활동 | "잠깐 스트레칭 하시는 건 어떠세요?" |
| 18:00 | 저녁 | "저녁 식사 시간이에요! 든든하게 드세요." |
| 21:00 | 취침 | "오늘 하루도 수고하셨어요. 편안한 밤 되세요." |

### 4.11 날씨 연동
| 항목 | 내용 |
|------|------|
| API | OpenWeatherMap (무료) |
| 캐시 | 30분 |
| 프론트엔드 | IdleScreen에서 10분마다 갱신 |
| 표시 | 아이콘 + 온도 + 상태 (한국어) |

### 4.12 야간 모드
| 항목 | 내용 |
|------|------|
| 기본 설정 | 22:00 ~ 07:00 |
| 동작 | 화면 어두운 톤 전환 (시계/날씨 저밝기) |
| 체크 주기 | 1분마다 자동 확인 |

### 4.13 보호자 인증
| 항목 | 내용 |
|------|------|
| 회원가입 | 이메일 + 비밀번호 (4자 이상) + 이름 |
| 비밀번호 | bcrypt 해싱 (복호화 불가) |
| 토큰 | JWT (72시간 유효) |
| 저장소 | Firestore 또는 로컬 JSON 파일 (폴백) |
| 대시보드 | 로그인 후 접근 가능, 로그아웃 지원 |

### 4.14 대화 기록 저장
| 항목 | 내용 |
|------|------|
| 저장 시점 | 세션 종료 시 (active → idle) |
| 저장 내용 | 대화 내역, 감정 리포트, 약 복용 여부, 긴급 여부 |
| 저장소 | Firebase Firestore `sessions` 컬렉션 |
| 조회 | `GET /api/sessions` + 대시보드에서 확인 |

### 4.15 앱 안정성
| 항목 | 내용 |
|------|------|
| WebSocket 보호 | JSON 파싱 try-catch |
| 카메라 복구 | 30프레임 실패 → 자동 재연결 |
| 마이크 복구 | 5회 재시도 + 화면 안내 |
| Firebase 안전 | 키 파일 없어도 크래시 없이 동작 |
| 헬스체크 | `GET /health` 엔드포인트 |
| 스레드 안전 | voice_agent 접근에 threading.Lock 적용 |
| Error Boundary | React 컴포넌트 에러 시 "다시 시작하기" 안내 |
| 스케줄러 안전 | 복약 알림 중복 방지 (replace_existing) |

## 8. 실행 방법

```bash
# 1. 백엔드
cd backend
pip install -r requirements.txt

cd frontend
npm install
npm run dev

# 3. 접속
# 시니어 앱: http://localhost:5173
```

## 9. 향후 계획

### 단기 (대학 프로젝트 완성)
- 실제 RPi5 배포 테스트
- 보호자 모바일 앱 (React Native)
- 실환경 STT/TTS 튜닝

- 복수 어르신 지원 (얼굴 인식 구분)
- MySQL/PostgreSQL 마이그레이션

- 두뇌 자극 게임
- 건강 데이터 연동 (웨어러블)
