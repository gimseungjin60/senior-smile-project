# AI-bum (시니어 스마일) 기능 명세 및 협업 가이드

본 문서는 프로젝트 협업자(기획자, 디저이너, 프론트엔드 등)에게 현재까지 백엔드(비전+음성)에서 구현된 **기능 스펙**과 **플로우**를 명확히 전달하기 위해 작성되었습니다.

---

## 🧭 1. 시스템 핵심 아키텍처
`AI-bum`은 노트북 및 라즈베리파이 환경에서 구동되는 **[ 비전 인식(OpenCV) + 다정한 대화 에이전트(LLM) + 스케줄러 ]** 기반의 지능형 케어 솔루션입니다.
*   웹캠 프레임에서 얼굴을 지속적으로 스캔합니다.
*   조건이 충족되면 시각적 오버레이와 동시에 백그라운드 대화 쓰레드가 가동됩니다.
*   프리리코딩된 아역 음성을 우선 재생하여 감성을 높이되, 상황에 따라 실시간 LLM 음성을 매끄럽게 엮어 출력합니다.

---

## ✨ 2. 현재 구현된 최신 기능 6선

### ① 비전-음성 자동 트리거 (Face Hooking)
카메라에 어르신이 3프레임 연속 인식되면 시스템 상태가 `idle`에서 `greeting`으로 전환됩니다. 이와 동시에 `VoiceAgent`가 깨어나 가장 선순위로 **귀가 인사**를 건넵니다.

### ② 어린이 음성 MP3 스마트 폴백 (Smart Audio Routing)
특정 상황 발생 시 값비싼 API 통신 대신 미리 준비된 폴더(`backend/sounds/`) 내의 아역 MP3를 최우선 재생합니다. 
*   **귀가 인사**: `greet_home.mp3`
*   **복약 알람**: `pill_remind.mp3`
*   **복약 칭찬**: `pill_praise.mp3` ("약, 먹었(묵었)" 키워드 파악 시)
*   **식사 안부**: `meal_check.mp3` ("밥, 식사" 키워드 파악 시)
> **💡 Fallback 시스템**: 지정된 오디오 파일이 실수로 지워져도 시스템이 뻗지 않고, 즉석에서 정해진 대사를 TTS로 변환하여 재생합니다.

### ③ 차분한 표준어 AI 페르소나 적용
일상적인 대화(위 4가지 조건 외)로 넘어가면 Gemini API (1.5 Flash)가 작동합니다. 시스템 프롬프트에는 *"예의 바르고 차분한 표준어를 사용하는 따뜻한 손주"* 라는 속성이 부여되어 있어 어르신에게 다정한 문장체로 응답합니다.

### ④ OpenCV 실시간 복약 오버레이
비전 스트리밍 화면(프론트엔드 모니터) 우측 하단에, 어르신의 "약 당일 복용 여부"가 오버레이 텍스트로 노출됩니다.
*   평상시: `<span style="color:red">[ Pill : NOT TAKEN ]</span>`
*   대화 중 약을 먹었다고 인지될 시: `<span style="color:green">[ Pill : COMPLETED ]</span>` 로 즉각 변경

### ⑤ 지능적 능동 알람 (Smart Scheduler)
가만히 대기하는 것을 넘어, 먼저 다가갑니다. `config.py`에 부여된 `PILL_TIME` (예: 09:00)이 되면 액자가 스스로 복약 독촉 멘트를 발생(`pill_remind.mp3`)시킵니다.
> **💡 눈치채기 로직**: 알람 시간이 되었지만, 만약 기기 앞에서 **어르신이 한창 다른 대화를 나누고 계신 상황이라면 알람을 조용히 '1분 연기'**하여 대화를 방해하지 않습니다.

### ⑥ 대화 이력 감정 1줄 요약 (AI Report)
어르신이 자리에서 벗어나 30초가 지나 대기 모드(`idle`)로 가거나, "들어가" 라고 말씀하셔서 시스템이 끊길 때 **그간 나눈 채팅 히스토리 전체가 내부적으로 제미나이에게 전송**됩니다.
*   서버 로그 출력 예시: `[AI 감정 리포트]: 할머니는 오늘 식사를 잘 마치셨고 쾌활하고 기분이 아주 좋으십니다.`

## 💻 3. 로컬 테스트 / 실행 가이드

프로젝트 저장소를 클론하신 후, 아래와 같이 로컬 서버를 띄워 모든 시나리오를 직접 테스트하실 수 있습니다.

```bash
# 1. 의존성 패키지 설치
pip install -r backend/requirements.txt

# 2. 백엔드 구동 (FastAPI + OpenCV)
cd backend
uvicorn main:app --reload

# * config.py 의 PILL_TIME 을 현재 시간 1~2분 뒤로 셋팅해두시면 알람 확인이 빠릅니다.
```
